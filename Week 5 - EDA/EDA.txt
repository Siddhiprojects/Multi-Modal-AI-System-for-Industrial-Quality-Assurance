## **Data Inspection and Validation**

### **When to Use**

- **New Dataset** Use `.head()`, `.info()`, `.describe()` to understand structure and potential issues.
- **Before Analysis** Check for missing data, inconsistent types, or incorrect values.

### **Core Commands**

```python

df.info()  # Data types and non-null counts
df.describe()  # Summary stats for numerical columns

```

```python

```

## **Handling Missing Data**

### **When to Use**

- **Small Missing Proportion (<5%)** Drop rows or columns
- **Moderate Missing Values** Impute with mean, median, or mode
- **Large Gaps** Use domain knowledge to impute or flag missing data

### **Strategies**

```python
df.dropna(thresh=len(df) * 0.95, axis=1, inplace=True)  # Drop columns with >5% missing
df["column"].fillna(df["column"].median(), inplace=True)  # Numerical imputation
df["category"].fillna(df["category"].mode()[0], inplace=True)  # Categorical imputation

```

## **Detecting and Managing Outliers**

### **When to Use**

- **Univariate Analysis** Use IQR or Z-Score methods.
- **High-Dimensional Data** Use machine learning methods like Isolation Forest.

### **Detection Techniques**

### **IQR**

**Why use IQR?** IQR is less sensitive to outliers than Z-Score.

```python
Q1 = df["Column"].quantile(0.25)
Q3 = df["Column"].quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
outliers = df[(df["Column"] < lower) | (df["Column"] > upper)]

```

### **Z-Score**

**Why use Z-Score?** Z-Score is sensitive to outliers but useful for standardized data.

```python
from scipy.stats import zscore
df["zscore"] = zscore(df["Column"])
outliers = df[df["zscore"].abs() > 3]

```

### **Management Techniques**

### **Log Transformation**

**Why to use log transformation?** Log transformation reduces the impact of outliers.
**When to use**: For right-skewed data.

```python
df["Log_Column"] = np.log(df["Column"] + 1)

```

### **Winsorization**

**Why use winsorization?** Winsorization replaces extreme values with less extreme ones.
**When to use**: For data with outliers

The choice between dropping and winsorizing outliers depends on your dataset and analysis goals. Drop outliers when you have sufficient data and the outliers are clearly errors or non-representative points, especially when your analysis requires strict normality. Use winsorization when working with smaller datasets where you can't afford to lose observations, or when the outliers represent extreme but legitimate values that you want to retain while reducing their impact on your analysis.

```python
from scipy.stats.mstats import winsorize
df["Winsorized_Column"] = winsorize(df["Column"], limits=[0.05, 0.05])

```

## **Hypothesis Testing**

### **When to Use**

- To validate assumptions or relationships in the dataset.

### **T-Test Example**

```python
from scipy.stats import ttest_ind
t_stat, p_val = ttest_ind(group1, group2)
```

In **machine learning (ML)**, a **t-test** is typically used in scenarios where you 
want to compare the performance of two models or algorithms. Here are a couple of key applications:

1. **Comparing Model Performance** – Suppose you have developed **two different models** for a classification task and want to know if the difference in their accuracy is 
**statistically significant**. You can run multiple experiments, collect accuracy scores from both models, and use a **t-test** to check whether 
one model consistently performs better.
2. **Feature Selection** – If you're analyzing whether a particular feature has a statistically significant effect on the target variable, a **t-test** can help 
determine whether to include or exclude a feature in the model.
3. **Evaluating Different Training Approaches** – When testing **two different training techniques**
 (e.g., different optimization algorithms, hyperparameter settings, or data preprocessing strategies), a **t-test** can be 
used to confirm whether a particular approach leads to a statistically significant improvement.
