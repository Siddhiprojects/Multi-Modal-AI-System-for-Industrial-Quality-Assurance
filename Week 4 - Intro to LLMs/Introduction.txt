LLMs are basically two files one file is parameters and the second file is to run the parameters code

An LLM is essentially:
Model Weights (Parameters): Learned knowledge stored as billions of numbers.
Inference Code: The program that loads these parameters and generates text
The intelligence is stored in the parameters, not in the code.

Training is much harder.
It involves:
Huge datasets (10+ TB text),Thousands of GPUs,Millions of dollars,Days or weeks of computation
Today's models (GPT-4, Claude) cost tens or hundreds of millions to train.

LLMs Learn by Predicting the Next Word
This is the most important concept.Predict the next word as accurately as possible.
Everything else emerges from this objective.

Why Next-Word Prediction Creates Intelligence
To predict the next word correctly, the model must learn:facts, grammar, history, science, coding, reasoning
language patterns.

LLM = Lossy Compression of the Internet
The speaker compares an LLM to:A compressed version of the Internet.
Unlike ZIP compression: LLM remembers concepts, forgets exact wording, recreates information
This explains why LLMs sometimes hallucinate.

LLMs sometimes invent information because they generate the most probable next word, not guaranteed truth.
Example: It may invent: ISBN numbers, names,dates,citations
Always verify important outputs.

Pretraining vs Fine-Tuning
Stage 1: Pretraining
Uses:Huge internet text
Goal:Learn knowledge.
Output:Base model.
Example:Wikipedia,Books,Forums,Research papers,Websites

Stage 2: Fine-Tuning
Uses:High-quality Question → Answer conversations written by humans.
Goal:Teach the model to behave like an assistant.
Output:ChatGPT-like assistant.

Easy way to remember
Pretraining → Learns information
Fine-tuning → Learns behavior

Reinforcement Learning from Human Feedback (RLHF)
Third optional stage.
Instead of writing answers,humans compare two responses.
Example:Answer A vs Answer B Human chooses better one. Model learns preferences.
Benefits:more helpful, safer, better aligned

Scaling Laws (Very Important)
One of the biggest ideas in AI.Performance depends mainly on:Number of parameters,
Amount of training data,Bigger model
More data
↓
Better performance
This has been consistently observed.

Closed Models

Calculator↓Solve math

Multimodal AI
Modern models can process multiple data types.
Examples:Text,Images,Audio,Video,Speech
Sketch website↓GPT generates HTML + CSS + JavaScript.

Context Window
Equivalent to the model's working memory.
The context window stores:conversation,uploaded files,previous prompts
Larger context window↓Better long-document understanding.

Retrieval-Augmented Generation (RAG)
Instead of relying only on memory,
the model searches:PDFs,documents,databases,company knowledge
Then generates answers.
Benefits:more accurate,updated information,less hallucination

System 1 vs System 2 Thinking
Current LLMs mostly behave like:
System 1:Fast,Instant,Pattern matching
Future AI aims for:
System 2:Slow reasoning,Planning,Reflection,Thinking before answering
This is an active research area.

Self-Improvement
Inspired by AlphaGo.Current LLMs imitate humans.
Future goal:Learn beyond humans.
Challenge:Unlike games,language has no simple reward function.

Custom GPTs:Users can build specialized assistants.
Uses:custom instructions,uploaded files,RAG

LLM as an Operating System
The speaker predicts future LLMs will become like an operating system.
Instead of managing hardware,they manage:browser,calculator,Python,files,APIs,memory,
image generation,speech,planning
This is the foundation of AI agents.

Security Risks (Important)
Jailbreaks,Users trick AI into ignoring safety.
Example:Pretend to be my grandmother...↓Model may answer restricted questions.
Prompt Injection
Hidden instructions inside:web pages,documents,images,can manipulate the AI.
Data Poisoning
Attackers insert malicious data into training.
Later,specific trigger words activate harmful behavior.
